# Contributing

Thanks for considering a contribution to Lingo Jungle.

## Ground Rules

- Keep pull requests focused and small when possible.
- Do not change core lesson or product logic unless the change fixes a clear bug.
- Update documentation when behavior, setup, or architecture changes.
- Match the existing code style and naming conventions.

## Setup

- Install frontend dependencies with `npm install` inside `frontend/`.
- Verify the frontend with `npm run lint` and `npm run build`.
- Verify the backend with `mvn test` inside `backend-java/`.

## Pull Requests

- Use a clear title and explain user-facing impact.
- Link related issues when relevant.
- Include screenshots or a short recording for UI changes.
- Mention any required environment variables or migration steps.

## Reporting Bugs

- Include reproduction steps.
- Include expected behavior and actual behavior.
- Share logs, screenshots, or console output when helpful.
